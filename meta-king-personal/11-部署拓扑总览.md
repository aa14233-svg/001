# 11 部署拓扑总览

> MetaKing 七体的物理部署 + 数据流。

## 七体节点

```
                         用户
                          │
                          ▼
                    ┌─────────────┐
                    │   Planner   │  ← 听证三步
                    │  (对话)      │
                    └──────┬──────┘
                           │ DAG
                           ▼
                    ┌─────────────┐
                    │   Thinker   │  ← 需求-环境对账
                    │  (探测)      │
                    └──────┬──────┘
                           │ 缺口清单
                           ▼
                    ┌─────────────┐
                    │ThinkerRepair│  ← 自修闸门
                    │  (执行)      │
                    └──────┬──────┘
                           │ (成功) 或 (失败 → Searcher)
                           ▼
                    ┌─────────────┐         ┌─────────────┐
                    │   Builder   │ ←──────│  Searcher   │
                    │  (执行)      │         │  (检索)      │
                    └──────┬──────┘         └─────────────┘
                           │ artifact
                           ▼
                    ┌─────────────┐
                    │   Auditor   │  ← 四维审查(横切拦截)
                    │  (拦截)      │
                    └──────┬──────┘
                           │ verdict
                           ▼
                    ┌─────────────┐
                    │  Recorder   │  ← 决策链持久化
                    │  (旁路监听)  │
                    └──────┬──────┘
                           │
                           ▼
                    ┌─────────────┐
                    │ PatternBank │  ← 三路径学习
                    │  (学习)      │
                    └─────────────┘
```

## 数据流

```
输入侧:
  用户消息 ──→ Planner (听证三步)
              ↓
  环境探测  ──→ Thinker (env_snapshot)
              ↓
  本地文件  ──→ Thinker (deps 探测)
              ↓
  外部 API  ──→ Searcher (按需)

执行侧:
  Planner DAG ──→ Thinker 对账表 ──→ ThinkerRepair 闸门 ──→ Builder
                                                              ↓
                                                       Auditor 四维审查
                                                              ↓
                                                       Recorder 持久化

存储侧:
  PostgreSQL:
    ├─ mechanism_endorsements    (新 M1-M7)
    ├─ pattern_bank_candidates   (候选池)
    ├─ pattern_bank_rules        (正式规则)
    ├─ causal_chains             (决策链)
    └─ recorder_history          (任务历史)

  Obsidian (本 vault):
    ├─ meta-king-personal/       (本文档集)
    └─ 人工智能入门/             (教材)
```

## 与老 KING 部署拓扑的对照

| 老 KING v3.0 | MetaKing v1.0 |
|-------------|---------------|
| 8 节点拓扑(主调度 + 6 子系统 + 双轨) | 7 体 + Auditor 横切 + PatternBank 旁路 |
| PostgreSQL 5 表(mechanism/function/data/causal/deployment) | PostgreSQL 5 表(mechanism/candidates/rules/causal/history) |
| 双轨查询(数理 → PG, 文档 → Obsidian) | 双轨保留,但 PG 侧重机制 + 学习,Obsidian 侧重设计文档 |
