# LSO-DAO 五境递进架构

> LSO-DAO 是 King Agent 的"修炼体系"——任务不是一次性完成，而是分五境递进，逐层深入。

## 一、五境递进总览

```
筑基 → 金丹 → 元婴 → 化神 → 渡劫
  │       │       │       │       │
  类型    安全    门控+   技能    内省+
  推导    扫描    域映射  路由    演化
                     │               验收
                    负载均衡
```

**核心哲学**：
- 任务不是"一次过"，而是"五层炼"
- 每一层都有明确的职责和出口标准
- 前一层未通过，不会进入下一层

**与三技能的关系**：
- master-engineer-hub 在"化神境"发挥路由决策
- cc-switch 在"化神境"提供 Provider 路由
- token-stats 在"渡劫境"执行 Token 审计

## 二、五境详解

### 第一境：筑基（类型推导）

**目标**：确定任务类型，建立基础认知

```
输入任务
  │
  ├── 任务类型识别
  │    ├── 代码任务 → 编码/开发域
  │    ├── 文档任务 → 文档/写作域
  │    ├── 设计任务 → 设计/创意域
  │    ├── 研究任务 → 学习/研究域
  │    ├── 管理任务 → 路由/调度域
  │    └── 混合任务 → 多域并行
  │
  ├── 上下文提取
  │    ├── 项目结构（当前项目文件树）
  │    ├── 依赖关系（package.json, requirements.txt 等）
  │    └── 历史记录（对话上下文、任务记录）
  │
  └── 基础计划生成
       ├── 任务分解 → 子任务列表
       ├── 资源估计 → 预估 Token 消耗
       └── 时间估计 → 预估执行时间
```

**出口标准**：
- [x] 任务类型已明确
- [x] 上下文已提取
- [x] 基础计划已生成

**示例**：

```
用户："在 React 项目中添加一个登录页面"

筑基境处理：
├── 任务类型：代码任务（前端开发）
├── 上下文提取：
│   ├── 项目结构：src/components, src/pages, src/api
│   ├── 依赖：react-router-dom, axios
│   └── 已有登录页面框架
└── 计划生成：
    ├── 子任务1：创建 LoginPage 组件
    ├── 子任务2：添加表单验证
    ├── 子任务3：对接登录 API
    └── 子任务4：添加路由配置
```

### 第二境：金丹（安全扫描）

**目标**：安全检查，确保任务在安全边界内执行

```
安全扫描
  │
  ├── 路径检查
  │    ├── 路径禁区防护（系统目录、敏感文件）
  │    │    └── 禁止：C:/Windows, /etc, .env
  │    ├── 系统文件保护（关键配置文件）
  │    │    └── 只读：package-lock.json, yarn.lock
  │    └── 配置只读保护（MCP 配置、技能配置）
  │         └── 只读：cc-switch 配置、token-stats 配置
  │
  ├── 高危操作拦截
  │    ├── 危险命令识别（rm -rf, format, drop table）
  │    ├── 网络操作审核（外部 API 调用、数据上传）
  │    └── 数据删除确认（文件删除、记录删除）
  │
  └── 权限验证
       ├── Agent 是否有权执行此操作？
       ├── 操作是否在预算范围内？
       └── 是否需要用户确认？
```

**出口标准**：
- [x] 安全扫描通过
- [x] 无高危操作
- [x] 权限已验证

**安全规则示例**：

```
路径禁区：
  - C:/Windows/**
  - /etc/**
  - .env
  - **/node_modules/**
  - **/.git/**

高危命令：
  - rm -rf / 或类似递归删除
  - format 磁盘操作
  - DROP TABLE 数据库删除
  - 未授权的网络请求

需确认操作：
  - 批量文件修改（>10 个文件）
  - 依赖安装/更新
  - 部署到生产环境
```

**目标**：门控决策 + 领域映射，确定执行路径

```
五行门控
  │
  ├── 金（代码）→ 编码/开发域
  │    ├── 前端开发 → FullStackAgent
  │    ├── 后端开发 → FullStackAgent
  │    └── 数据库 → DataAgent
  │
  ├── 木（文档）→ 文档/写作域
  │    ├── 技术文档 → DocAgent
  │    ├── 商业文档 → DocAgent + doc-writing-guide
  │    └── 创意写作 → ArtisticDesignerAgent
  │
  ├── 水（数据）→ 数据/分析域
  │    ├── 数据分析 → DataAgent
  │    ├── 数据可视化 → DataAgent + ArtisticDesignerAgent
  │    └── 数据清洗 → DataAgent
  │
  ├── 火（设计）→ 设计/创意域
  │    ├── UI/UX 设计 → ArtisticDesignerAgent
  │    ├── 架构设计 → ProgramDesignerAgent
  │    └── 视觉设计 → ArtisticDesignerAgent
  │
  └── 土（部署）→ 部署/运维域
       ├── CI/CD 配置 → DeployAgent
       ├── 环境部署 → DeployAgent
       └── 监控配置 → DeployAgent + RtkAgent

域映射
  └── 门控结果 → 映射到 8 大技能域
       ├── 金 → 路由域 + 编码域
       ├── 木 → 文档域
       ├── 水 → 数据域
       ├── 火 → 设计域 + 创意域
       └── 土 → 部署域 + 安全域
```

**出口标准**：
- [x] 五行门控已确定
- [x] 域映射已完成
- [x] 对应技能组已激活

### 第四境：化神（技能路由 + 负载均衡）

**目标**：路由到具体技能 + 均衡负载

```
技能路由
  │
  ├── 在激活域中匹配技能
  │    ├── master-engineer-hub 路由
  │    │    ├── Level 0/1：0 token 直通
  │    │    └── Level 3：LLM 兜底 → 机械降神
  │    │
  │    ├── cc-switch Provider 路由
  │    │    ├── Provider 可用性检查
  │    │    └── Provider 切换执行
  │    │
  │    └── token-stats Token 预算检查
  │         ├── 本月已消耗 Token
  │         ├── 预算剩余（月上限 $15）
  │         └── 是否触发告警
  │
  └── 负载均衡
       ├── 当前 Provider 负载检查
       │    ├── API 响应时间
       │    ├── 队列深度
       │    └── 错误率
       │
       ├── Token 预算剩余检查
       │    ├── 预算充足 → 使用高性能 Provider
       │    └── 预算紧张 → 切换便宜 Provider
       │
       └── 最优 Agent 选择
            ├── Agent 可用性
            ├── Agent 专长匹配
            └── Agent 负载
```

**出口标准**：
- [x] 技能已匹配
- [x] Provider 已选择（考虑成本和负载）
- [x] Agent 已分配

### 第五境：渡劫（内省 + 演化 + 验收）

**目标**：自我验证 + 经验积累 + 质量验收

```
内省（反射）
  │
  ├── 执行过程检查
  │    ├── 是否偏离计划？
  │    ├── 是否有错误？
  │    └── 是否需要调整？
  │
  ├── 演化（学习）
  │    ├── 记录成功模式 → 引力盆
  │    │    └── 如："切换 Provider" 成功 → 引力盆 hit+1
  │    ├── 记录失败模式 → 避免重复
  │    │    └── 如：Provider 不可用 → 标记避免下次选择
  │    └── 更新技能路由权重
  │         └── 成功技能权重 +0.1，失败技能权重 -0.1
  │
  └── 验收（D1/D2/D3）
       ├── D1：阶段验收
       │    ├── 当前阶段任务全部完成
       │    ├── 产出物经 Agent 自查
       │    └── 无重大错误或偏离
       │
       ├── D2：模块验收
       │    ├── 相关模块的所有子任务完成
       │    ├── 模块间接口正常
       │    └── 模块功能符合预期
       │
       └── D3：总体验收
            ├── 全任务所有子任务完成
            ├── 所有模块集成正常
            ├── 最终产出符合原始需求
            ├── 所有 Hook 检查均通过
            └── Token 消耗在预算内
```

**出口标准**：
- [x] D1/D2/D3 全部通过
- [x] Token 审计完成
- [x] 经验已记录到引力盆
- [x] 结果已交付

**验收不通过处理**：

```
D1 不通过：
  └── 最多 3 次修正循环
  └── 每次记录失败原因
  └── 3 次后升级到 D2 审查

D2 不通过：
  └── 最多 2 次修复循环
  └── 2 次后升级到 D3 审查

D3 不通过：
  └── 重大问题 → 人工介入
  └── 记录失败模式到引力盆（避免重犯）
```

## 三、五境与 System Prompt 的对应关系

| 五境 | 对应 System Prompt 层 | 加载时机 | 加载内容 |
|------|---------------------|---------|---------|
| 筑基 | 基础 System Prompt | 任务初始化 | 角色定义、安全约束、工作原则 |
| 金丹 | 安全规则（core-rules） | 安全扫描 | 路径禁区、高危命令、权限规则 |
| 元婴 | 增强版 System Prompt | 门控决策 | 五行门控、域映射逻辑 |
| 化神 | SkillHub + SKILL.md | 技能路由 | 技能定义、工具列表、路由规则 |
| 渡劫 | 验收规则 + 引力盆 | 完成时 | 验收标准、学习机制、Token 审计 |

## 四、呼吸管道

五境递进运行在"呼吸管道"之上，确保任务可暂停、可恢复、可清理：

```
吸入（吸）→ 任务到达，准备资源
  │         ├── 分配 Agent
  │         ├── 加载 System Prompt
  │         └── 准备工具集
  │
储存（存）→ 状态缓存，上下文持久化
  │         ├── 任务状态保存
  │         ├── 中间结果缓存
  │         └── 上下文序列化
  │
呼出（呼）→ 执行任务，产出结果
  │         ├── 五境逐层执行
  │         ├── Hook 检查
  │         └── 产出中间/最终结果
  │
化合（化）→ 结果整合，数据关联
  │         ├── 多 Agent 结果合并
  │         ├── 数据交叉验证
  │         └── 最终格式转换
  │
归墟（归）→ 资源释放，临时数据清理
  │         ├── Agent 工具从视野移除
  │         ├── 临时文件清理
  │         ├── Token 审计记录
  │         └── 引力盆更新
```

**呼吸管道的核心价值**：

| 特性 | 说明 | 示例 |
|------|------|------|
| 可暂停 | 任务可在任意阶段暂停 | 用户中断后恢复执行 |
| 可恢复 | 状态持久化，恢复时从暂停点继续 | 断网恢复后任务继续 |
| 可清理 | 完成后自动清理临时资源 | 临时文件、缓存、Agent 工具 |
| 可追溯 | 每个阶段的状态都可追溯 | 审计日志、Token 消耗记录 |

| 五境 | 参与的 Agent | 负责 | 典型工具 |
|------|-------------|------|---------|

## 六、五境递进的完整示例

```
用户："帮我写一个 Python 脚本，读取 Excel 数据并生成图表"

筑基境：
├── 类型识别：代码任务 + 数据处理
├── 上下文：用户项目中有 data.xlsx
└── 计划：读取 → 分析 → 绘图 → 输出

金丹境：
├── 路径检查：data.xlsx 可读
├── 高危操作：无（只读数据）
└── 权限验证：通过

元婴境：
├── 五行门控：金（代码）+ 水（数据）
└── 域映射：编码域 + 数据域

化神境：
├── 技能路由：omnipotent-engineer
├── Provider 选择：当前 Provider（预算充足）
└── Agent 分配：FullStackAgent + DataAgent

渡劫境：
├── D1：读取成功，分析完成
├── D2：图表生成，符合预期
├── D3：全任务完成，交付脚本
├── Token 审计：~200 token
└── 引力盆更新：f("excel图表") + 1
```

## 补充说明

### 1. 五境与三大技能的深度集成

| 五境 | master-engineer-hub | cc-switch | token-stats |
|------|-------------------|-----------|-------------|
| 筑基 | 路由前准备 | Provider 状态检查 | 预算初始检查 |
| 金丹 | 安全路由规则 | MCP 安全配置 | 审计规则加载 |
| 元婴 | 门控路由决策 | Provider 域映射 | 成本域映射 |
| 化神 | **核心路由执行** | **Provider 切换** | **预算实时检查** |
| 渡劫 | 路径固化到引力盆 | 切换日志记录 | **Token 审计完成** |

### 2. 呼吸管道的性能指标

| 呼吸阶段 | 平均耗时 | 内存占用 | 数据量 |
|---------|---------|---------|--------|
| 吸入 | <5ms | ~100KB | ~2KB (System Prompt) |
| 储存 | <2ms | ~50KB | 任务状态 |
| 呼出 | 取决于任务 | 取决于 Agent | 产出结果 |
| 化合 | <10ms | ~30KB | 合并结果 |
| 归墟 | <5ms | 释放 | 清理数据 |

### 3. 异常处理

当五境中某一境失败时：

| 失败境界 | 处理策略 | 回退方案 |
|---------|---------|---------|
| 筑基 | 重新提取上下文 | 默认使用"通用任务"类型 |
| 金丹 | 记录安全违规 | 拒绝执行，返回错误信息 |
| 元婴 | 降级到单域执行 | 默认使用路由域 |
| 化神 | 切换到备用 Provider | 自动回退 Level 3 兜底 |
| 渡劫 | 触发 Loop 循环 | 3 次失败后人工介入 |

### 4. 设置建议

| 环境 | 建议配置 | 说明 |
|------|---------|------|
| 开发环境 | 五境全开，严格验收 | 确保质量 |
| 测试环境 | 开启筑基→化神，渡劫简化 | 快速验证 |
| 生产环境 | 五境全开，全流程监控 | 质量+安全 |
| 紧急修复 | 跳过金丹→元婴，直通化神 | 快速响应 |
