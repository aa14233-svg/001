## 三、拓扑层次总结

| 层次 | 名称 | 核心组件 | 职责 |
|------|------|---------|------|
| L1 | 入口层 | TraeOrchestrator + System Prompt | 接收请求，角色定义 |
| L2 | Hook 层 | 4 个 Hook 节点 | 安全约束，质量检查 |
| L3 | 路由层 | master-engineer-hub + 引力盆 | 0 token 路由 A/B/C |
| L4 | 执行层 | 20+ Agent 集群 | 具体任务执行 |
| L5 | 基础设施 | LSO-DAO 五境递进 | 修炼体系，递进执行 |
| L6 | 技能层 | SkillHub + 8 大技能组 | 专业能力提供 |
| L7 | 验收层 | D1/D2/D3 + Loop | 质量验收，迭代改进 |

## 四、数据流路径

```
用户输入（自然语言）
  │
  ├── ● 路由路径（高频 90%）
  │     引力盆/特征词典命中 → 直通对应 Agent → 执行 → 返回
  │     Token 成本：0
  │
  ├── ● 标准路径（中频 8%）
  │     LLM 兜底 → 路由 A/B/C → 五境递进 → SkillHub → 验收 → 返回
  │     Token 成本：中等
  │
  └── ● 扩展路径（低频 2%）
       LLM 兜底 → 能力缺口 → 机械降神 → GitHub 检索 → 注入 → 执行 → 消散
       Token 成本：高
```
