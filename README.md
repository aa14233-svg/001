# 🧠 第二篇：King-Skill 技能群系系统

## 什么是 King Agent？

**King Agent** 是 Trae IDE 中运行的**一体超强智能体**，它不是单一模型，而是一个 **智能体操作系统**——以 TraeOrchestrator 为主调度器，协调 20+ 专业 Agent 集群、6 大核心技能系统、五境递进基础设施层。

## 核心哲学

> **不持一物，而万物皆备。不预一器，而万器待召。**

## 系统架构（7层拓扑）

```
用户输入
   │
   ▼
┌────────────────────────────────────────────────┐
│ L1  入口层：TraeOrchestrator + System Prompt    │
├────────────────────────────────────────────────┤
│ L2  Hook层：安全检查 → 资源确认 → 依赖检查       │
├────────────────────────────────────────────────┤
│ L3  路由层：master-engineer-hub（假借修真）      │
│      Level 0/1/2/3 → A/B/C  Agent 视野隔离      │
├────────────────────────────────────────────────┤
│ L4  执行层：20+ Agent 集群                      │
├────────────────────────────────────────────────┤
│ L5  基础设施：LSO-DAO 五境递进 + 呼吸管道        │
├────────────────────────────────────────────────┤
│ L6  技能层：SkillHub 8 大技能组                  │
├────────────────────────────────────────────────┤
│ L7  验收层：Hook 3+4 + D1/D2/D3 + Loop          │
└────────────────────────────────────────────────┘
   │
   ▼
交付结果
```

## 三大核心技能

### 1️⃣ master-engineer-hub（路由编排引擎）

> **不持一物，而万物皆备。不预一器，而万器待召。**

- 假借修真：高频请求 0 token 直通（90%）
- 机械降神：按需从 GitHub 注入能力
- 引力盆：动态固化路由模式
- Agent 视野隔离：A/B/C 各看各的工具

### 2️⃣ cc-switch（Provider 路由 + MCP 管理）

> AI 提供商的"交换机"

- 10 个 MCP 工具：provider 切换、MCP 同步、技能同步
- 与 token-stats 联动：自动切换最低成本 Provider

### 3️⃣ token-stats（Token 审计 + 预算管理）

> AI 的"财务总监"

- 6 个 MCP 工具：查询、对比、预算、审计、报告、告警
- 月预算 $15 上限，超额自动预警

## Hook+Loop 工程约束

- **4 个 Hook 节点**：准备 → 执行中 → 验收 → 完成
- **三级验收**：D1（阶段）→ D2（模块）→ D3（总体验收）
- **Loop 循环**：不通过 → 修正 → 重验（最多3次）
